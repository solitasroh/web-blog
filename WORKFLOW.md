# 블로그 포스팅 워크플로우

## 프로세스

### 1단계: 경험 공유 (마스터 → AI)
**마스터가 공유**:
- 주제 선정
- 실무 경험, 겪은 문제
- 해결 과정, 핵심 기술
- 코드 스니펫, 예시
- 강조하고 싶은 포인트

**방법**:
- 텔레그램으로 자유롭게 공유
- 길이 제한 없음 (짧아도 OK, 길어도 OK)
- 코드는 파일 경로 제공 or 직접 붙여넣기

---

### 2단계: 초안 작성 (AI)
**AI가 작성**:
- MDX 포맷 블로그 포스트
- Frontmatter (title, date, tags, excerpt)
- 도입부 → 본론 → 마무리 구조
- 코드 예시 포함
- 읽기 시간 5-15분 분량

**전달 방법**:
- MDX 파일 텔레그램 전송
- 또는 GitHub 브랜치 생성 → PR

---

### 3단계: 검토 & 수정 (마스터)
**마스터가 확인**:
- 기술적 정확성
- 톤 앤 매너
- 추가/삭제할 내용
- 오타, 문법

**피드백 방법**:
- "여기 수정해줘" → AI가 반영
- "이 부분 추가" → AI가 추가
- "OK, 배포" → 다음 단계

---

### 4단계: 배포 (AI)
**배포 프로세스**:
1. `apps/blog/content/posts/` 에 MDX 파일 추가
2. `git add`, `git commit`, `git push origin main`
3. Vercel 자동 배포 (2-3분 소요)
4. 배포 URL 텔레그램 공유

---
